# Инструкция по настройке базы данных

## Что уже сделано:

1. ✅ Создана Prisma схема с моделями:
   - User (пользователи)
   - Track (треки)
   - Playlist (плейлисты)
   - PlaylistTrack (связь треков и плейлистов)
   - Favorite (избранное)

2. ✅ Создан Prisma Client в `src/lib/prisma.ts`

3. ✅ Настроен .env файл с DATABASE_URL

4. ✅ Применена миграция к базе данных

5. ✅ Установлены зависимости:
   - prisma
   - @prisma/client
   - bcryptjs
   - @types/bcryptjs

6. ✅ Созданы API endpoints:
   - POST /api/auth/register - регистрация
   - POST /api/auth/login - вход
   - GET /api/tracks - получить все треки
   - POST /api/tracks - создать трек
   - GET /api/tracks/[id] - получить трек
   - DELETE /api/tracks/[id] - удалить трек
   - GET /api/users/[id] - получить пользователя
   - PATCH /api/users/[id] - обновить пользователя

7. ✅ Обновлены компоненты для работы с API:
   - AuthContext - регистрация и вход через API
   - Страница загрузки треков - сохранение в БД
   - Главная страница - загрузка треков из БД
   - Профиль - обновление аватара через API

## Как запустить:

1. Убедитесь, что PostgreSQL запущен на localhost:5432

2. Запустите dev сервер:
   ```bash
   npm run dev
   ```

3. Откройте http://localhost:3000

## Как использовать:

1. Зарегистрируйте нового пользователя - он будет добавлен в базу данных
2. Войдите в систему
3. Загрузите трек - он будет сохранен в базу данных
4. Все треки загружаются из базы данных

## Проверка базы данных:

Чтобы посмотреть данные в базе:
```bash
npx prisma studio
```

Это откроет веб-интерфейс для просмотра и редактирования данных.
