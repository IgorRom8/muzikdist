### 2. Реструктуризация по FSD

#### Шаг 1: Создать структуру папок
```bash
mkdir -p src/pages/{home,auth,player,upload,favorites}
mkdir -p src/widgets/{header,sidebar,player,track-list}
mkdir -p src/features/{auth/{login,register,logout},track/{play-track,upload-track,favorite-track}}
mkdir -p src/entities/{user,track,album,playlist}/{model,ui,api}
mkdir -p src/shared/{ui/{button,input,card},lib/{hooks,utils},api,config,types}
```

#### Шаг 2: Переместить компоненты

**Widgets:**
- `src/components/Header.tsx` → `src/widgets/header/ui/Header.tsx`
- `src/components/Sidebar.tsx` → `src/widgets/sidebar/ui/Sidebar.tsx`
- `src/components/Player.tsx` → `src/widgets/player/ui/Player.tsx`

**Features:**
- Логика авторизации → `src/features/auth/`
- Логика плеера → `src/features/track/play-track/`

**Entities:**
- `src/types/index.ts` → `src/entities/{user,track}/model/types.ts`
- `src/lib/mockData.ts` → `src/entities/track/model/mockData.ts`
- `src/lib/auth.ts` → `src/entities/user/api/authService.ts`

**Shared:**
- Общие UI компоненты → `src/shared/ui/`
- Context → `src/shared/lib/context/`

#### Шаг 3: Обновить импорты

Настроить алиасы в `tsconfig.json`:
```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"],
      "@/app/*": ["./src/app/*"],
      "@/pages/*": ["./src/pages/*"],
      "@/widgets/*": ["./src/widgets/*"],
      "@/features/*": ["./src/features/*"],
      "@/entities/*": ["./src/entities/*"],
      "@/shared/*": ["./src/shared/*"]
    }
  }
}
```

#### Шаг 4: Создать публичные API

Каждый модуль должен экспортировать через `index.ts`:
```typescript
// src/widgets/header/index.ts
export { Header } from './ui/Header'

// src/features/auth/login/index.ts
export { LoginForm } from './ui/LoginForm'
export { useLogin } from './model/useLogin'
```

### 3. Запуск проекта

После всех изменений:
1. Удалить папку `.next` вручную
2. Запустить `npm run dev`
3. Проверить все страницы
