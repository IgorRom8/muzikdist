# Исправление ошибки Access Denied

## Проблема
```
AccessDenied: Access Denied
```

Это значит, что S3 credentials неправильные или у них нет прав.

## Решение: Создайте новый ключ доступа в Selectel

### Шаг 1: Откройте Selectel
1. Зайдите на https://my.selectel.ru/
2. Войдите в аккаунт
3. Перейдите в **Облачная платформа** → **Объектное хранилище**

### Шаг 2: Создайте новый ключ
1. В левом меню найдите **"Ключи доступа"** или **"Access Keys"**
2. Нажмите **"Создать ключ"** или **"Create key"**

### Шаг 3: Настройте права
Выберите права для ключа:
- ✓ **Чтение** (Read) - обязательно
- ✓ **Запись** (Write) - обязательно
- ✓ **Удаление** (Delete) - опционально

### Шаг 4: Выберите bucket
- Выберите bucket: **s3-muzik**
- Или выберите "Все bucket'ы" если хотите универсальный ключ

### Шаг 5: Создайте и скопируйте
1. Нажмите **"Создать"**
2. **ВАЖНО:** Скопируйте оба ключа:
   - **Access Key ID** (например: `abc123def456...`)
   - **Secret Access Key** (например: `xyz789uvw012...`)
3. Сохраните их в безопасном месте - Secret Key показывается только один раз!

### Шаг 6: Обновите в Vercel
1. Откройте https://vercel.com/dashboard
2. Выберите проект `muzikdist`
3. Settings → Environment Variables
4. Найдите и обновите:
   - `AWS_ACCESS_KEY_ID` = новый Access Key ID
   - `AWS_SECRET_ACCESS_KEY` = новый Secret Access Key
5. Нажмите **Save**

### Шаг 7: Обновите локально (опционально)
Обновите файл `.env`:
```env
AWS_ACCESS_KEY_ID=ваш_новый_access_key
AWS_SECRET_ACCESS_KEY=ваш_новый_secret_key
```

### Шаг 8: Redeploy
1. В Vercel: Deployments → последний деплой → ... → **Redeploy**
2. Попробуйте загрузить файл снова

## Проверка

После redeploy попробуйте загрузить файл. Должно работать!

Если все еще ошибка:
1. Проверьте что bucket `s3-muzik` существует
2. Проверьте что регион правильный: `ru-7`
3. Проверьте что ключи скопированы без пробелов в начале/конце

## Альтернатива: Проверьте существующий ключ

Если не хотите создавать новый:
1. Найдите существующий ключ с ID: `ecd2796237554e8eab8f3672a6a67f32`
2. Проверьте его права - должны быть Read + Write
3. Проверьте что он активен (не заблокирован)
4. Проверьте что он привязан к bucket `s3-muzik`
